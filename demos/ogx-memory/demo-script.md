# OGX + MemoryHub Demo Script

Zero-code persistent memory for an OGX agent. The agent discovers and calls
MemoryHub tools through OGX's MCP tool runtime. No `import memoryhub` anywhere.

## Setup

```bash
# 1. Set environment variables
export MEMORYHUB_MCP_URL="https://memory-hub-mcp-memory-hub-mcp.apps.<cluster>/mcp"
export MEMORYHUB_API_KEY="mh-dev-..."
export TOGETHER_API_KEY="..."

# 2. Start OGX with MemoryHub tools registered
llama stack run ./run.yaml

# 3. In another terminal, start a client session
llama stack run client
```

## Session 1: The agent learns

**Presenter says:** "This agent has never seen this user before. Watch it store
preferences to MemoryHub via tool calls."

```
User: I'm setting up a new microservices project. A few things about how I
      work: I always use Podman, never Docker. All containers must use Red Hat
      UBI base images. And I need FIPS compliance for crypto.

Agent: (calls register_session, then write_memory for each preference)
       Got it. I've recorded your preferences:
       - Container runtime: Podman (not Docker)
       - Base images: Red Hat UBI only
       - Crypto: FIPS compliance required

User: Also, we use FastAPI for all Python services and pytest for testing.

Agent: (calls write_memory for each)
       Noted. FastAPI for services, pytest for testing.
```

**Presenter highlights:**
- The agent called MemoryHub tools directly through OGX's tool runtime
- Each `write_memory` call is visible in the turn as a tool invocation
- Zero memory code in the application. The `run.yaml` is the only integration point.

## Session 2: The agent remembers

**Presenter says:** "New session. The agent has no conversation history. But
watch what happens when we ask it to do something."

```
User: Help me containerize this FastAPI app.

Agent: (calls register_session, then search_memory)
       Based on your preferences, here's a Containerfile using Podman with
       a UBI 9 base image and FIPS-compliant crypto:

       FROM registry.redhat.io/ubi9/python-312:latest
       ...
       # Built for podman build --platform linux/amd64

User: Nice. And set up the test suite.

Agent: (may search_memory again for testing preferences)
       Setting up pytest with your project structure...
```

**Presenter highlights:**
- The agent was never told about Podman, UBI, or FIPS in this session
- It searched MemoryHub and found the preferences from session 1
- This works because MemoryHub is infrastructure, not application code
- Any MCP-capable agent framework gets the same capability

## Key talking points

1. **Zero application code.** The only integration is a `run.yaml` entry. No SDK,
   no imports, no memory-handling logic.

2. **Governed memory.** Unlike local-first solutions, these memories are stored
   with RBAC, audit trails, and scope isolation. An admin can see what the agent
   remembered, quarantine bad memories, or enforce retention policies.

3. **Framework-agnostic.** This demo uses OGX, but the same MCP server works with
   Claude Code, Codex CLI, kagenti, or any MCP-capable tool. Generate onboarding
   instructions for your framework: `memoryhub config init --format ogx|system-prompt|agents-md`.

4. **Multi-tenant ready.** Multiple users and teams share the same MemoryHub
   instance with scope isolation. One user's preferences are invisible to another's.

5. **The agent decides what to remember.** MemoryHub provides the storage and
   governance. The agent's system prompt (generated by `memoryhub config init`)
   guides when to read and write. No rigid schema, no forced extraction.

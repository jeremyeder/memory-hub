# OGX + MemoryHub: Zero-Code Agent Memory

This demo shows an OGX (LlamaStack) agent getting persistent cross-session
memory from MemoryHub without any memory-handling code in the application.

## How it works

1. `run.yaml` registers MemoryHub's MCP server as a tool group
2. OGX discovers the available tools (`register_session`, `memory`, `thread`)
3. The agent's system prompt (generated by `memoryhub config init --format ogx`)
   tells it when to read and write memories
4. Memories persist in MemoryHub's governed PostgreSQL store across sessions

## Prerequisites

- OGX (llama-stack) installed with MCP support
- A running MemoryHub instance (MCP server endpoint)
- A MemoryHub API key
- A Together AI API key (or swap the inference provider in `run.yaml`)

## Quick start

```bash
# Set credentials
export MEMORYHUB_MCP_URL="https://memory-hub-mcp-memory-hub-mcp.apps.<cluster>/mcp"
export MEMORYHUB_API_KEY="mh-dev-..."
export TOGETHER_API_KEY="..."

# Generate agent instructions for OGX
memoryhub config init --format ogx --non-interactive

# Start OGX
llama stack run ./run.yaml
```

See [demo-script.md](demo-script.md) for the full walkthrough.

## What makes this different

| Concern | Without MemoryHub | With MemoryHub |
|---|---|---|
| Memory persistence | None (context resets each session) | Governed, multi-tenant PostgreSQL |
| Integration effort | Build your own | One line in `run.yaml` |
| Access control | N/A | OAuth 2.1, RBAC, scope isolation |
| Audit trail | None | Structured audit on every operation |
| Framework lock-in | Tied to one tool | Works with any MCP-capable agent |
